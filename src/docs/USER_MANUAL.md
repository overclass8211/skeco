## 1. 시스템 개요

OCI CRM AI는 **태양광·EPC 영업 조직**을 위한 통합 CRM + AI 어시스턴트 시스템입니다.

### 🎯 핵심 특징

| 영역 | 기능 |
|------|------|
| **영업관리** | 리드 파이프라인, 고객사, 프로젝트, 활동 이력 |
| **AI 어시스턴트** | Gemini 기반 챗봇, 고객사 브리핑, 회의록 자동화 |
| **회의록 STT** | 음성 → 텍스트 (최대 120분, 화자분리) |
| **Google 연동** | Calendar, Meet, Gmail (읽기/발송/자동 매칭) |
| **모바일 PWA** | 오프라인 녹음 + 자동 동기화 |
| **다국어** | 한국어 / 영어 / 일본어 / 중국어 |
| **권한 제어** | 5단계 RBAC (매니저 → Superadmin) |

### 🏗 기술 스택 (개발자 참고)

- **백엔드**: Node.js + Express + MariaDB
- **인증**: JWT (Access 15분 / Refresh 7일) + bcrypt + TOTP 2FA + WebAuthn
- **프론트엔드**: Vanilla JS SPA (프레임워크 없음)
- **AI**: Google Gemini 2.5 Flash/Pro
- **PWA**: Service Worker + IndexedDB
- **실시간**: WebSocket (공지사항, 헬스맵)
- **외부**: Google OAuth 2.0 (Calendar/Meet/Gmail)

## 2. 시작하기

### 2.1 로그인

1. 브라우저에서 시스템 URL 접속
2. **사용자명(또는 이메일) + 비밀번호** 입력
3. 2FA 활성화 시 → OTP 6자리 추가 입력
4. 로그인 성공 후 자동으로 대시보드로 이동

### 2.2 2FA (2단계 인증) 설정 — 권장

위치: **사용자 설정 (우측 상단 프로필 메뉴)**

1. 설정 > "2FA 활성화" 클릭
2. **Google Authenticator** 같은 OTP 앱으로 QR 코드 스캔
3. 화면에 표시되는 6자리 코드 입력 → 검증 완료
4. 다음 로그인부터 OTP 입력 필수

### 2.3 비밀번호 변경

설정 > "비밀번호 변경" → 현재 비밀번호 + 새 비밀번호 입력

### 2.4 세션 관리

- **Access Token**: 15분 자동 만료
- **Refresh Token**: 7일 (HttpOnly 쿠키, 자동 갱신)
- 로그아웃 시 모든 토큰 무효화

## 3. 주요 화면 가이드

### 3.1 📊 대시보드

**경로**: 사이드바 첫 번째 메뉴 (홈 아이콘)

**표시 정보**:
- 활성 리드 수 / 입찰 진행건 / 올해 수주액 / 수주율
- 월별 신규 리드 추이 (차트)
- 파이프라인 펀넬 (단계별 건수 + 금액)
- 지역별(국내/해외) 현황
- 최근 활동 타임라인
- 🔔 마감 임박 / 입찰 마감 / 오늘 일정

**연도 필터**: 우측 상단에서 연도 변경 가능

---

### 3.2 📈 영업 파이프라인 (칸반 보드)

**경로**: 사이드바 > "영업 파이프라인"

**8단계 파이프라인**:
```
lead → review → proposal → bidding → negotiation → won
                                                  ↘ lost
                                                  ↘ dropped
```

**사용법**:
- 카드를 **드래그&드롭**으로 단계 이동
- 카드 클릭 시 리드 상세 모달 오픈
- 필터: 지역, 사업유형, 담당자, 기간

---

### 3.3 🎯 영업 리드

**경로**: 사이드바 > "영업 리드"

**핵심 필드**:
- 고객사명 + 프로젝트명
- 사업유형 (태양광/모듈/EPC/ESS/전기/설치)
- 지역 (국내/해외) + 용량(MW)
- 예상 금액 + 통화
- 단계 + 담당자
- **예상마감일** (수주 예상 일자)
- **입찰마감일** (입찰서 제출 데드라인)
- 메모

**기능**:
- ✅ 단건 등록 / 수정 / 삭제
- ✅ **대량 등록**: Excel에서 복사 → "Copy & Paste" 영역에 붙여넣기
- ✅ **내보내기**: Excel/CSV 다운로드
- ✅ **활동 이력**: 리드별 미팅/통화/이메일 추적
- ✅ **Gmail 자동 매칭**: 고객 이메일 주소로 송수신 메일 자동 표시

---

### 3.4 🏢 고객사

**경로**: 사이드바 > "고객사"

**카드형 목록 화면**:
- 고객사명 + 산업군 + 지역 표시
- 리드 수 / 활성 딜 / 수주 누계 미니 통계

**상세 모달 (4탭)**:
1. **📋 정보·수정**: 기본 정보 (담당자, 연락처, 이메일, 주소, 카카오맵)
2. **🤝 관련 딜**: 해당 고객사의 모든 리드
3. **✨ 핵심 브리핑**: **AI 자동 생성** 영업 인사이트
4. **👥 소속 고객**: 담당자 목록

**📩 최근 Gmail 대화**: 고객 이메일 주소와의 메일 자동 매칭

---

### 3.5 📦 프로젝트 (수주 후 관리)

**경로**: 사이드바 > "프로젝트"

**핵심 필드**:
- 프로젝트명, 고객사, 유형
- **계약 금액** + **예상 원가** + **마진율** (자동 계산)
- 상태: 진행중 / 제조중 / 납기지연 / 완료 / 취소
- 납기일, 담당자
- 연결된 리드 (lead_id)

**기능**: CRUD + 대량 등록 + 마진 분석

### 3.6 📅 캘린더 (영업 일정)

**경로**: 사이드바 > "영업 캘린더"

**FullCalendar 기반**:
- 월/주/일 뷰 전환
- 이벤트 드래그&드롭으로 일정 변경
- 색상별 구분 (이벤트 타입)
- Google Calendar **양방향 동기화**

**필터**: 담당자별 / 리드별 / 기간

---

### 3.7 🎙 회의록 (STT + AI 요약)

**경로**: 사이드바 > "회의록 AI" + "회의록 목록"

#### 회의록 작성 흐름

**3가지 입력 방법**:
1. **🔴 실시간 녹음**: 페이지에서 직접 녹음 (최대 120분)
2. **📁 파일 업로드**: MP3/WAV/M4A/WEBM/OGG (최대 25MB)
3. **📱 오프라인 녹음**: PWA에서 오프라인 녹음 → 온라인 복귀 시 자동 동기화

#### 처리 방식
- **짧은 녹음** (~20분): 동기 처리 (`/transcribe`)
- **긴 녹음** (20~120분): 비동기 처리 (`/transcribe-async` + 폴링)
- **자동 화자분리**: "스피커1", "스피커2" 라벨링
- **AI 요약**: 핵심 내용 + 액션 아이템 자동 추출

#### Google Meet 통합
- 페이지 내 "📹 Google Meet 연동" 섹션
- "🔗 Meet 링크 생성" → 즉시 미팅 생성
- 생성된 미팅 링크는 캘린더에 자동 등록

## 4. AI 기능

### 4.1 🤖 AI 어시스턴트 (챗봇)

**위치**: 우측 상단 "🪄 AI 어시스턴트" 버튼

**사용 시나리오**:
- "이번 분기에 입찰 마감 임박한 리드 알려줘"
- "우리 팀의 수주율은?"
- "고객사 A의 최근 활동을 요약해줘"

**특징**:
- **스트리밍 응답** (한 글자씩 출력)
- **컨텍스트 자동 주입**: 현재 CRM 통계 + 최근 리드 + 입찰 일정
- **한국어 우선** 답변
- 사용 모델: Gemini 2.5 Flash

---

### 4.2 ✨ 고객사 AI 브리핑

**위치**: 고객사 상세 모달 > "✨ 핵심 브리핑" 탭

**자동 분석 항목**:
- 고객사 기본 정보 + 산업 동향
- 진행 중인 리드 요약
- 최근 활동 패턴
- 추천 영업 액션

**사용 모델**: Gemini 2.5 Pro (정확도 우선)

---

### 4.3 🎙 회의록 자동 작성

회의록 페이지에서 자동으로 다음을 수행:

1. **음성 → 텍스트** (화자 분리 포함)
2. **AI 요약 생성**:
   - 회의 주제
   - 핵심 논의 포인트
   - 액션 아이템 (담당자/기한 포함)
3. **마크다운 형식**으로 저장

---

### 4.4 🪙 AI 토큰 관리

- 사용자별 **월간 한도** (기본 500,000 토큰)
- 임계값 도달 시 **자동 충전** 가능 (관리자 설정)
- 설정 페이지에서 본인 사용량 확인 가능
- 관리자는 사용자별 한도 + 충전 이력 관리

## 5. Google 연동

### 5.1 최초 연결 방법

1. **설정 페이지** → "Google 계정 연결"
2. 팝업으로 Google OAuth 동의 화면 표시
3. 다음 4가지 권한 허용:
   - ✅ Google Calendar (읽기/쓰기)
   - ✅ Google Meet (자동 활성)
   - ✅ **Gmail 읽기** (`gmail.readonly`)
   - ✅ **Gmail 발송** (`gmail.send`)
4. 팝업 자동 닫힘 → 메인 화면에서 ✅ 연결 완료 확인

### 5.2 📅 Google Calendar

**기능**:
- CRM 이벤트 ↔ Google Calendar 양방향 동기화
- 이벤트 생성 시 자동으로 Google Calendar에 반영
- Webhook으로 Google 변경사항 자동 수신

---

### 5.3 📹 Google Meet

**회의록 페이지에서**:
- "🔗 Meet 링크 생성" 버튼 클릭
- 미팅 제목 + 시작 시간 + 소요 시간 입력
- 즉시 Meet 링크 생성 + Calendar 자동 등록
- 최근 생성된 미팅 목록 표시 (📝 회의록 연결 상태 포함)

---

### 5.4 📧 Gmail 통합 (Phase G1 + G2 + G3)

#### G1: Gmail 읽기 + 자동 매칭
- **고객사 모달**: "📩 최근 Gmail 대화" 섹션 자동 표시
- **리드 상세**: 담당자 이메일로 자동 매칭
- 송수신 메일 최근 10건 표시

#### G2: Gmail 직접 발송
- 이메일 작성 모달에 **"📧 Gmail 로 발송"** 버튼
- CRM 내부에서 직접 메일 발송
- 발송 시 활동 이력에 자동 기록 (제목, 내용 포함)
- 한국어 제목 안전 처리 (RFC 2047 인코딩)

#### G3: 백그라운드 자동 동기화
- **회의록 페이지 하단**: "📧 Gmail 자동 동기화" 토글
- 활성화 시 **5분 주기**로 자동 폴링
- 새 메일이 고객 이메일과 매칭되면 → **활동 이력에 자동 기록**
- 매칭되지 않은 개인 메일은 기록하지 않음 (프라이버시 보호)
- "⚡ 지금 동기화" 버튼으로 즉시 수동 실행 가능

#### 보안
- 토큰 AES-256 암호화 저장
- 권한 만료 시 (`invalid_grant`) 자동 비활성화 + 재연결 안내
- 재연결 시 stale 에러 메시지 자동 클리어

## 6. 모바일 / PWA

### 6.1 PWA 설치 방법

**iOS (Safari)**:
1. 시스템 URL 접속
2. 공유 버튼 (📤) > "홈 화면에 추가"
3. 홈 화면에 OCI CRM 아이콘 생성

**Android (Chrome)**:
1. 시스템 URL 접속
2. 메뉴 (⋮) > "홈 화면에 추가" 또는 "앱 설치"
3. 네이티브 앱처럼 사용 가능

### 6.2 모바일 최적화

- 16px 폰트 (iOS 자동 zoom 방지)
- 햄버거 메뉴 자동 표시
- 하단 네비게이션 (PWA 모드)
- 상단 타이틀 가로 줄바꿈 방지

### 6.3 📱 오프라인 회의록 녹음

**시나리오**: 출장 중 네트워크 없는 상황에서 녹음

1. 회의록 페이지에서 **녹음 시작** (오프라인이어도 가능)
2. 녹음 완료 → **IndexedDB 큐에 저장** (자동)
3. 인터넷 연결 복귀 시 → **자동 업로드 + STT 처리**
4. 상태 확인:
   - `pending` (대기) → `uploading` (업로드중) → `transcribing` (변환중) → `done` (완료)
   - 실패 시 `error` 상태 + 재시도 가능

**제약**:
- 브라우저 IndexedDB 용량: iOS ~50MB / Chrome ~수GB

### 6.4 Service Worker 캐싱 전략

| 리소스 | 전략 |
|--------|------|
| HTML | Network-First (실패 시 `/offline.html` 폴백) |
| CSS / JS / 이미지 / 폰트 | Cache-First (속도 우선) |
| `/api/*` | No Cache (실시간 데이터) |
| `/uploads/*` | No Cache |

## 10. FAQ / 문제해결

### Q1. 로그인이 안 됩니다.
- 비밀번호 재설정: 관리자에게 요청
- 2FA 코드 오류: OTP 앱 시간 동기화 확인

### Q2. Google 연결이 끊겼다고 나옵니다.
1. 설정 > "Google 연결 해제"
2. 다시 "Google 연결" 클릭 → 권한 재허용
3. 자동으로 옛 에러 메시지가 정리됩니다

### Q3. 회의록 녹음이 504 에러로 실패합니다.
- 20분 이상의 긴 녹음은 자동으로 **비동기 처리**됩니다
- 페이지를 닫지 마시고, 진행 상태 확인 후 완료까지 대기

### Q4. 오프라인에서 녹음한 파일이 업로드되지 않습니다.
- 인터넷 연결 확인
- 회의록 페이지 진입 시 자동 동기화 시작
- IndexedDB 큐가 가득 찼을 가능성 (관리자 문의)

### Q5. Gmail 자동 동기화가 동작하지 않습니다.
1. 회의록 페이지 > "📧 Gmail 자동 동기화" 토글 ON
2. "⚡ 지금 동기화" 클릭하여 즉시 테스트
3. 매칭 0건 = 정상 (해당 고객 이메일로 주고받은 메일 없음)
4. "재연결 필요" 표시 → Google 연결 해제 후 재연결

### Q6. AI 토큰이 부족하다고 나옵니다.
- 설정 > 토큰 사용량 확인
- 관리자에게 한도 증액 요청 또는 자동충전 활성화

### Q7. 라벨을 변경했는데 즉시 적용되지 않습니다.
- 브라우저 새로고침 (F5)
- 캐시 만료 (최대 10분) 후 자동 반영

### Q8. 카드 드래그가 안 됩니다.
- 모바일에서는 길게 누른 후 드래그
- 권한 확인 (담당 리드만 변경 가능)

### Q9. 고객사 모달에서 "Unknown column 'contact_email'" 에러가 나옵니다.
- 2026.05 버전에서 해결됨 (`customers.email` 컬럼 사용으로 수정)
- 운영 서버 업데이트 후 PM2 재시작 필요

### Q10. PWA 설치 후 업데이트가 반영되지 않습니다.
- Service Worker 캐시 버전 자동 갱신
- 강제 새로고침: 브라우저에서 캐시 삭제 후 재방문

### Q11. AI 제안 분석/평가가 "분석 진행이 안됨" 상태로 멈춥니다.
- 파일 형식 확인: **PDF / 이미지(PNG·JPG·WEBP) / 텍스트** 만 분석 가능
- PPT/DOC/HWP/XLS 는 PDF로 변환 후 재업로드
- 운영 .env 에 `GEMINI_API_KEY` 설정 여부 관리자 확인
- 브라우저 콘솔(F12)에 상세 에러 출력됨

### Q12. 공유 링크가 만료되었다고 합니다.
- 공유 발급 시 기본 7일 (선택: 14/30일/무제한)
- 모달 → [📤 발송 & 이력] 탭 → [🔁 재발급] 클릭 시 새 토큰 발급
- 기존 링크는 즉시 무효화

## ⏰ Phase 4 — 만료 알림 자동화 (v5.9.4 신규)

> **자동화**: 매일 오전 9시 KST 에 만료 임박 알림 자동 처리
> **위치**: 계약 편집 모달 하단 "⏰ 만료 알림" 섹션
> **2회 알림**: D-`renewal_notice_days` (기본 30일) + D-7 (최종 경고)

### 🚀 사용자 가치

```
계약 종료일 ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━ 만료일
                  ↑                  ↑
              renewal_notice      D-7 (최종 경고)
              (기본 30일)
                  ↓                  ↓
              in-app + email     in-app + email
              자동 알림            자동 알림
```

### 🛠 동작 흐름

1. **계약 생성/수정** → `end_date` 입력 → 알림 자동 enqueue (best-effort)
2. **계약 편집 모달** → 알림 섹션에서 예정/발송/취소 이력 확인
3. **cron (매일 09:00 KST)** → scheduled_for ≤ today 인 pending 처리 → status=sent
4. **계약 상태가 terminated/expired** → pending 알림 자동 cancel

### 📊 알림 상태 3가지

| 상태 | 의미 | 표시 |
|------|------|------|
| `pending` | 발송 대기 | 🔵 예정 (취소 버튼 표시) |
| `sent` | 발송 완료 | ✅ 발송완료 (sent_at 표시) |
| `cancelled` | 취소됨 | ⚪ 취소됨 (회색) |

### 🔔 알림 채널

#### 1. in-app (항상 활성)
- `contract_alerts` 테이블이 큐 역할
- 계약 모달에서 직접 조회
- 별도 UI 알림 (예: 토스트) 없음 — 사용자가 모달 열어서 확인하는 방식

#### 2. email (옵션 — `.env CONTRACT_ALERT_EMAIL_ENABLED=1` 시)
- 계약 owner_id 의 Gmail OAuth 토큰 사용 → 본인에게 자가 알림 발송
- 토큰 미연동 시 skip (in-app 만 동작, 에러 없음)
- 발송 실패해도 in-app 큐는 sent 갱신 (안정성)

### ⏰ 알림 시점 — 사용자 커스터마이즈

계약별 `renewal_notice_days` 필드 (기본 30) 로 1차 알림 시점 조정:

| 설정 | 1차 알림 | 2차 알림 (고정) |
|------|---------|---------------|
| `renewal_notice_days=30` | D-30 | D-7 |
| `renewal_notice_days=60` | D-60 | D-7 |
| `renewal_notice_days=14` | D-14 | D-7 |
| `renewal_notice_days=7` | D-7 (중복 — 1건만) | — |

### 🔄 자동 동작

| 트리거 | 동작 |
|--------|------|
| 계약 생성 (POST `/`) | end_date 있으면 자동 enqueue (2건) |
| 템플릿 적용 (POST `/from-template/:id`) | end_date 있으면 자동 enqueue |
| 계약 수정 (PUT `/:id`) — end_date 변경 | pending 모두 cancel 후 재 enqueue |
| 계약 수정 — renewal_notice_days 변경 | 동일 (1차 시점 재계산) |
| 계약 수정 — end_date 비움 | pending 모두 cancel |
| 상태 전이 → terminated | pending 모두 cancel |
| 상태 전이 → expired | pending 모두 cancel |

### 🛠 관리자 도구

- `GET /api/contracts/:id/alerts` — 계약별 알림 조회 (pending+sent+cancelled)
- `DELETE /api/contracts/alerts/:alertId` — 개별 알림 취소 (pending → cancelled)
- `POST /api/contracts/alerts/process` — 큐 수동 처리 (cron 트리거 대신, 테스트용)

### 💌 이메일 발송 활성화 절차

1. `.env` 에 `CONTRACT_ALERT_EMAIL_ENABLED=1` 추가
2. 계약 owner 가 Google OAuth (Gmail 연동) 완료되어 있어야 함
3. PM2 restart → 콘솔에 `[contractAlerts] ... email=on` 확인
4. cron 트리거 시 owner 의 Gmail 로 본인에게 자가 알림 발송

### 안전성 보장

- 알림 처리 실패해도 계약 생성/수정 성공 (best-effort)
- email 발송 실패해도 in-app 큐는 sent 갱신
- 테스트 환경 (`NODE_ENV=test`) 에서는 cron 미등록
- 한 번에 최대 500건 처리 (큐 폭주 방지)

현재 상태에 따라 모달 footer 에 자동으로 표시되는 액션 버튼:

| 현재 상태 | 빠른 액션 |

### 🛡 자동 동작

- **`signing → active`**: `start_date` 가 비어있으면 오늘 날짜 자동 채움 + Toast 안내
- **`* → expired`**: history 에 `⏰ 만료 처리: 발효 → 만료` 강조
- **`* → terminated`**: history 에 `❌ 해지 처리: ⃝⃝ → 해지` 강조, 이후 어떤 액션도 불가
- **`active → renewal`**: history 에 `🔄 갱신 시작` 강조
- **`renewal → active`**: history 에 `🔄 갱신 완료` 강조

### ⚠️ 만료 임박 표시 (목록)

`status='active'` 이고 `end_date` 가 **오늘부터 30일 이내**인 계약은 목록의 종료일 컬럼 아래에 빨간색으로:
```
2026.06.10
⚠️ D-18 만료 임박
```

D-N 은 종료일까지 남은 일수.

### 🔄 잘못된 전이 시도 시

- 백엔드가 400 응답 + 친절한 한국어 메시지:
  ```
  잘못된 전이: 초안 → 발효 (허용: 검토중, 해지)
  ```
- Toast 로 표시되어 사용자에게 즉시 알림

### 🔧 하위 호환 — 관리자 우회

- PUT `/api/contracts/:id` 로 status 변경 시 **전이 검증 안 함**
- 데이터 마이그레이션, 관리자 직접 수정 시 우회 경로 보존
- 정상 워크플로우는 PATCH `/api/contracts/:id/status` 만 사용

> **비용**: 1회 약 500-1000원 (Gemini Pro API 토큰)
> **소요 시간**: 30-60초

### 사용 흐름

1. 계약 편집 모달 진입 → 첨부 파일 영역
2. 분석 가능 파일 (PDF / PNG / JPG / WEBP / TXT / MD) 옆 **[🤖 법무]** 버튼 클릭
3. confirm 다이얼로그 — 비용/시간 인지 후 진행
4. 30-60초 대기 → 결과 카드 즉시 표시 (스크롤 자동 이동)
5. **모달 닫고 다시 열어도 최신 검토 결과 자동 표시** (`latest_legal_review` prefill)

### 신뢰성 보장 (v5.9.1 환각 가드)

- **risk_level ↔ review_score 일관성 강제**: AI 가 두 값을 어긋나게 반환하면 score 기준으로 자동 보정 + 콘솔 경고
