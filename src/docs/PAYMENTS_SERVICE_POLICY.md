## 2. 데이터 모델 정책

- **Model A (평면/비정규화)**: 계약 단위 정보(고객사·계약명·통화·계약기간)를 각 단계 행에 비정규화 저장.
- **마스터 연결은 선택적**(느슨한 결합, FK 없음): `contract_id` / `customer_id`가 채워지면 연결, 비면 "직접 등록(계약 없음)".
- **표시 신뢰값 우선**: 계약이 연결되면 `contracts.title` / `contracts.customer_name`(신뢰값)으로 표시 → 비정규화 드리프트 차단. 미연결이면 비정규화값 사용.

## 3. 수금 일정 = 매출 계획

### 상태 전이
```
scheduled(예정) → invoiced(청구) → partial(부분수금) → collected(수금완료)
                                     ↘ overdue(연체)        ↘ written_off(대손)
```

### 규칙
- `scheduled_amount` = **VAT 포함** 금액.
- 통화: **KRW 기본** + 9종(USD/JPY/EUR/GBP/CNY/AUD/SGD/HKD/VND) 허용.
- **계약 → 자동생성**: 비율 템플릿 기반 → 금액 = 총액 × 비율, 예정일 = 기준일 + offset_days.
- **날짜 검증(서버 방어)**:
  - 계약 시작/종료일 연도 4자리
  - 수금예정일 ≥ 계약 시작일
  - 단계 순서: 착수금 ≤ 중도금 ≤ 잔금
- **저장**: 1 계약 모달 → N행을 **1 트랜잭션**(create/upsert/delete 원자 처리).

---

## 4. 입금 = 수금 실적

- `payment_records`: 1 스케줄 → N 입금. **입금 시 상태 자동 재계산** (수금합 ≥ 예정 → collected, > 0 → partial).
- **은행 거래내역 자동 매칭** (무스키마, `payment_records` 재사용):
  - 입력: 은행 CSV/엑셀/붙여넣기 → 컬럼 매핑(입금일 · 입금액 · 입금자명 · 적요)
  - 매칭 점수: 잔액 정확일치 **+100** / 예정액 일치 **+90** / 부분(amt≤잔액) **+40**, 입금자명 정규화 일치 **+50**·포함 **+30**, 예정일 ±3일 **+20**·±14일 **+10**
  - confidence: ≥120 높음(🟢) / ≥60 보통(🟡) / 그외 낮음(⚪)
  - 확정 → 입금 일괄 등록(트랜잭션) + 상태 자동전환

---

## 5. 세금계산서

- 상태: `draft`(작성중) → `requested`(발행요청) → `issued`(발행완료) → `cancelled`(취소)
- **자동 연동**: `issued` 전환 시 `issued_at` 기록 + 연결 수금 스케줄 `status='invoiced'` 자동전환.
- **삭제 정책**: 발행완료(issued) 건 삭제 차단(draft만 삭제 가능).
- **홈택스 가져오기**: CSV/xlsx 위치기반 파싱 → 컬럼 매핑 → `issued` 일괄 등록(**승인번호 중복 스킵**).
- 바로빌 자동발행/국세청 전송: 상용 API 키 등록 후 진행(현재 수동 기록).

---

## 6. 연체 미수금 자동 알림

- **일일 스캔**: due_date 경과 + scheduled/invoiced → `overdue` 동기화 → 잔여 미수금 있는 건만 대상.
- **알림 생성**: 건당 1회 인앱(dedup `overdue:<id>`) + **재무팀 메일**(설정값, 하루 1건 dedup).
- **메일 발송**: Gmail OAuth 우선, 미연결 시 `pending` 큐잉(가짜 발송 안 함).
- **스케줄러**: 부팅 90초 후 + 매일(운영 env). 수동 스캔 버튼 제공.
- **재무팀 메일 설정**: 미수금 탭에서 입력 → `system_settings.payment_overdue_notify_email` (이메일 형식 검증, 빈 값=해제).

---

## 7. 계약별 기준 통일 + 정합성

| 단계 | 내용 |
|---|---|
| 표시 통일 | 수금현황 **1열=계약(명+번호)** · **2열=고객사** 분리 |
| 신규 등록 정합 | 수금 모달 고객사 입력 → customers 마스터 **typeahead** 선택 시 `customer_id` 연결(직접 입력=free-text 허용) |
| 정합성 인사이트 | 계약금액(VAT포함) vs 수금계획 합 → **▲미편성 / ✓계획일치 / ▼초과** 배지(0.5%·1만원 임계) |

## 8. UI / 표시 정책

- 탭: **수금현황 · 미수금 · 세금계산서 · 매출분석**
- 뷰: 계약별 그룹(기본) ↔ 전체 평면 토글(localStorage), 그룹 정렬 = 다음 수금예정일.
- 매출분석: Chart.js 3종(월별 예정vs실적 · 상태별 도넛 · 연체 TOP5) + 손익 시뮬레이터.
- 엑셀 내보내기: 현재 필터 반영(.xlsx).
- **모달**: CSP(`script-src-attr`)가 인라인 `onclick` 차단 → `Modal.open` **클릭 위임**으로 close 버튼 CSP-safe 처리(동적 footer 포함).

# 🧪 테스트 시나리오 (직접 테스트용)

> 전제: `crm.payments` 활성 + team_lead 이상 로그인 + 수금관리 메뉴 진입.
> 표기: ✅ 통과 / ❌ 실패 — 각 단계의 **예상 결과**를 기준으로 판정.

## TC-01 — 수금 스케줄 등록 (고객사 마스터 연결)
**절차**
1. `[+ 수금 스케줄 등록]` 클릭
2. 고객사 입력란에 등록된 고객사명 2글자+ 입력
3. 드롭다운 제안에서 고객사 선택
4. 수금 단계 1개 추가(단계명·예정일·금액) 후 저장

**예상 결과**
- [ ] 고객사 입력 시 customers 마스터 자동완성 제안이 뜬다
- [ ] 선택 시 입력란이 채워진다(내부적으로 `customer_id` 연결)
- [ ] 저장 성공 토스트, 수금현황 목록에 새 행 표시

## TC-02 — 계약에서 자동 생성
**절차**
1. `[📄 계약에서 생성]` 클릭 → 계약 검색·선택 → 비율 템플릿 선택 → 미리보기 → 생성
**예상 결과**
- [ ] 템플릿 비율대로 단계별 금액·예정일이 자동 산출된다
- [ ] 생성 후 해당 계약 그룹이 수금현황에 묶여 표시된다

## TC-03 — 입금 등록 → 상태 전환
**절차**
1. 수금현황에서 한 행의 `💳`(입금등록) 클릭 → 금액/입금일 입력 → 저장
**예상 결과**
- [ ] 입금합 < 예정액 → 상태 `부분수금`
- [ ] 입금합 ≥ 예정액 → 상태 `수금완료`
- [ ] 수금 상세의 "실제 수금액"이 갱신된다

## TC-04 — 은행 거래내역 자동 매칭
**전제**: `mock-data/은행거래내역_샘플_32건.xlsx` 또는 실제 은행 거래내역 파일
**절차**
1. 헤더 `[🏦 은행 거래내역]` → `📄 파일` 선택 → 분석 → (컬럼 자동매핑 확인) → 자동 매칭
2. 매칭 테이블에서 🟢 건 확인 → `입금 등록`
**예상 결과**
- [ ] 컬럼(입금일·입금액·입금자명·적요)이 자동 매핑된다
- [ ] 미수금과 금액·입금자명·예정일로 매칭(🟢/🟡/⚪)된다
- [ ] 입금 등록 시 해당 스케줄에 입금 기록 생성 + 상태 자동전환
- ⚠️ 실데이터 사용 시 등록은 실제 수금 기록을 남긴다(테스트는 등록 직전까지 권장)

## TC-05 — 세금계산서 발행요청 → 발행완료 → 수금 자동연동
**절차**
1. 수금 상세 → `🧾 세금계산서 발행요청` → 저장(작성중)
2. 세금계산서 탭에서 해당 건 → `발행완료(issued)` 전환
**예상 결과**
- [ ] 발행요청 생성 시 status=작성중
- [ ] issued 전환 시 발행일시 자동 기록 + **연결 수금 스케줄이 `청구(invoiced)`로 자동 전환**
- [ ] 발행완료 건은 삭제 버튼이 차단된다

## TC-06 — 홈택스 세금계산서 가져오기
**절차**
1. 세금계산서 탭 → `[⬆ 홈택스 가져오기]` → 파일/붙여넣기 → 분석 → 매핑 → 가져오기
**예상 결과**
- [ ] 일괄 등록(issued) 결과(건수/중복 스킵/오류)가 토스트로 표시된다
- [ ] **승인번호 중복 건은 자동 스킵**된다

## TC-07 — 연체 미수금 알림
**절차**
1. 미수금 탭 → `📧 재무팀 알림 메일` 입력 → `저장`
2. `↻ 지금 스캔` 클릭
3. `🔔 연체 알림` 클릭 → 목록 확인 → `모두 읽음`
**예상 결과**
- [ ] 메일 형식이 잘못되면 저장 시 에러
- [ ] 스캔 후 "연체 N건 · 신규 알림 …" 토스트
- [ ] 알림 목록에 연체 건(고객사·D+N·금액) 표시, 미읽음 배지 갱신
- [ ] 같은 연체 재스캔 시 중복 알림이 생기지 않는다(dedup)

## TC-08 — 계약별 정합성(미편성/계획일치)
**전제**: 계약금액이 입력된 계약에 연결된 수금 일정
**절차**
1. 수금현황(계약별 그룹뷰)에서 해당 계약 그룹 행 확인
**예상 결과**
- [ ] 수금예정액 아래 `계약금 ₩X` 표시
- [ ] 계약금액 > 수금계획 → `▲ 미편성 ₩차액`(빨강)
- [ ] 계약금액 = 수금계획 → `✓ 계획일치`(초록)

## TC-09 — 수금현황 그룹/평면 뷰
**절차**
1. `[📂 계약별]` ↔ `[☰ 전체]` 토글, 부모행 클릭(펼침/접힘)
**예상 결과**
- [ ] 그룹뷰: 계약(부모) + 단계(자식) 2단, 1열 계약 / 2열 고객사
- [ ] 평면뷰: 행별 계약·고객사·단계 분리 표시
- [ ] 뷰 선택이 새로고침 후에도 유지(localStorage)

## TC-10 — 엑셀 내보내기
**절차**: 수금현황 `⤓ 엑셀` 클릭
**예상 결과**
