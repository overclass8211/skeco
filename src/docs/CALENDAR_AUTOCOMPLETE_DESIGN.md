> **작성일**: 2026-05-18
> **대상**: 디자이너, 기획, 개발자
> **상태**: 설계 단계 (코드 미구현)

## 1. 개요

### 1.1 배경
영업 캘린더의 **새 일정 등록 모달**에서 고객사 입력 시:
- 현재: 자유 텍스트 입력 (오타/표기 차이 발생)
- 영업기회 드롭다운: 전체 리드 표시 (관련 없는 딜까지)
- 단일 담당자, 정적 UI

### 1.2 목표
1. **고객사 자동완성** — 2자 이상 입력 시 기존 데이터 기반 추천
2. **고객사 ↔ 영업기회 자동 연결** — 고객사 선택 시 관련 활성 딜만 필터
3. **데이터 무결성** — `customer_id` 자동 매핑 (텍스트 → FK)
4. **점진적 UX 개선** — 시간/참석자/회의록/반복 일정 단계적 도입

### 1.3 측정 가능한 효과
- 고객사 입력 시간: ~10초 → ~2초 (80% 단축)
- 오타/표기 차이로 인한 데이터 중복: ~15% → 거의 0
- 영업기회 연결률: ~30% → 70%+ (예상)

## 5. 인터랙션 명세

### 5.1 자동완성 트리거
| 조건 | 동작 |
|------|------|
| 0~1자 입력 | 드롭다운 표시 안 함 |
| 2자 이상 입력 | 250ms 디바운스 후 API 호출 |
| Backspace로 2자 미만 | 드롭다운 자동 닫힘 |
| ESC 키 | 드롭다운 닫힘 (입력값 유지) |
| 외부 클릭 | 드롭다운 닫힘 |

### 5.2 키보드 네비게이션
| 키 | 동작 |
|----|------|
| ↓ | 다음 항목 하이라이트 |
| ↑ | 이전 항목 하이라이트 |
| Enter | 선택 (없으면 첫 항목) |
| Tab | 선택 + 다음 필드로 이동 |
| Esc | 드롭다운 닫힘 |

### 5.3 선택 후 동작
1. 입력 필드: 선택한 고객사명 표시
2. Hidden field: `customer_id` 저장
3. 영업기회 드롭다운: API 재호출 (`?customer_id={id}`)
4. 부가 정보 풍선: 산업/지역/딜 수 표시
5. 초기화 버튼: 선택 해제 + 영업기회 드롭다운도 리셋

### 5.4 모바일 동작
- 드롭다운: 키보드 위에 표시 (z-index 최상위)
- 터치 친화: 항목 높이 44px+ (Apple HIG)
- 결과 항목 시각화 약간 단순화 (모바일 가독성)

## 6. API 명세 (기존 확장)

### 6.1 `GET /api/customers` 확장 (자동완성 전용 모드)

**Request**:
```
GET /api/customers?search=삼성&limit=10&autocomplete=1
```

**신규 파라미터**:
- `autocomplete=1` — 자동완성 모드 활성
  - Smart Ranking 적용
  - 응답에 부가 정보 (`active_deals_count`, `is_my_customer`) 포함

**Response**:
```json
{
  "success": true,
  "data": [
    {
      "id": 5,
      "name": "삼성케미칼",
      "industry": "에너지",
      "region": "국내",
      "country": "대한민국",
      "contact_person": "박팀장",
      "active_deals_count": 3,
      "is_my_customer": true,
      "last_activity_at": "2026-05-10T14:00:00Z",
      "match_score": 95
    }
  ],
  "total": 12,
  "query": "삼성"
}
```

**Smart Ranking 알고리즘**:
```
score = base
      + (exact_match ? 50 : starts_with ? 30 : 10)
      + (is_my_customer ? 20 : 0)
      + (active_deals_count * 5)
      + (last_activity < 30d ? 15 : 0)
```

### 6.2 영업기회 필터링 (기존 활용)

```
GET /api/leads?customer_id=5&active_only=1&limit=20
```

→ 이미 `/api/leads` 가 `customer_id`, `stage` 필터 지원함. `active_only=1` 만 추가하면 됨 (`stage NOT IN ('won','lost','dropped')`).

### 6.3 신규 고객사 등록 (기존 API)

```
POST /api/customers
{ "name": "신규ABC", "region": "국내", "industry": "..." }
```

→ 자동완성 드롭다운의 "+ 새 고객사 등록" 클릭 시 이 API 호출.

## 7. DB 영향

### 7.1 calendar_events 테이블 강화 (선택)

**현재**:
```sql
customer_name VARCHAR(200)  -- 자유 텍스트
```

**Phase 1 후 권장**:
```sql
customer_id   INT NULL,       -- 신규 컬럼
customer_name VARCHAR(200),   -- 표시용 유지 (FK breaking 방지)
FOREIGN KEY (customer_id) REFERENCES customers(id) ON DELETE SET NULL
```

**마이그레이션**:
```sql
ALTER TABLE calendar_events
  ADD COLUMN customer_id INT NULL AFTER lead_id,
  ADD INDEX idx_customer_id (customer_id);
```

→ idempotent ALTER (initTables.js 패턴)

### 7.2 영향 없음 옵션
- DB 스키마 변경 없이 `customer_name` 만 정확히 저장도 가능
- 향후 분석 위해 `customer_id` 권장

## 10. 디자이너 협의 포인트

### 10.1 디자인 결정 필요 항목
1. **드롭다운 항목 디자인**
   - 카드형 vs 라인형
   - 아이콘 (🏢 vs SVG)
   - 진행딜 수 표시 위치/스타일
   - 본인담당 배지 색상/디자인

2. **부가 정보 풍선 (선택 후)**
   - 위치 (input 아래 vs 옆)
   - 폰트 크기
   - 색상 (강조 vs 보조)

3. **신규 등록 옵션**
   - 드롭다운 마지막 항목 vs 별도 영역
   - 색상 (CTA vs 보조)

4. **모바일 UX**
   - 드롭다운 풀스크린 vs 인라인
   - 터치 영역 크기

5. **시간 선택기 (Phase 3)**
   - Native input[type=time] vs 커스텀
   - 24시간제 / 12시간제

### 10.2 인터랙션 결정
1. 디바운스 시간 (250ms 권장, 협의 가능)
2. 최소 입력 문자 수 (2자 권장)
3. 검색 결과 최대 개수 (10개 권장)
4. ESC 시 입력값 유지 여부 (유지 권장)

### 10.3 접근성 (WCAG 2.1)
- 키보드만으로 모든 동작 가능 (필수)
- 스크린 리더 친화 (aria-* 속성)
- 색상 외에도 정보 표현 (텍스트/아이콘)

- 백엔드: `search` 인덱스 활용 (`customers.name` 인덱스 존재 확인 필요)

### 11.2 데이터 정합성
**위험**: 자유 텍스트 입력 시 customer_id 누락
**완화**:
- 드롭다운에서 선택 안 한 경우 — 신규 등록 유도
- 또는 자유 입력도 허용하되 customer_id NULL 유지 (현재 패턴)
- 캘린더는 가벼운 일정 도구라 강제하지 않음

### 11.3 UX 학습 곡선
**위험**: 기존 사용자 혼란
**완화**:
- placeholder: "예: 삼성케미칼 (2자 이상 입력 시 추천)"
- 첫 사용 시 toast: "💡 고객사명 일부만 입력해도 자동 추천됩니다"

### 11.4 모바일 키보드 충돌
**위험**: 드롭다운이 가상 키보드에 가려짐
**완화**:
- viewport 이벤트로 키보드 감지
- 드롭다운을 input 아래 → 위로 이동 (자동)
