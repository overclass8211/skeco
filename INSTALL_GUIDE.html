👋 시작하기 전에

이 가이드는 프로그래밍을 처음 접하는 분도 따라할 수 있도록 만들어졌습니다. 한 단계씩 천천히 진행하시면 됩니다. 막히면 맨 아래 "막혔을 때" 섹션을 참고하세요.

💡 이 시스템이 하는 일: 내 컴퓨터를 작은 서버로 만들어서, 같은 컴퓨터(또는 같은 사내망의 다른 컴퓨터)에서 웹브라우저로 CRM에 접속할 수 있게 합니다.

운영체제 확인

  • macOS 사용자 → "macOS" 탭을 따라가세요.
📌 필요한 사양 보통의 사무용 PC면 충분합니다. (메모리 4GB↑, 저장공간 2GB↑)

Node.js 설치

Node.js는 이 CRM 프로그램을 실행시키는 엔진입니다.

2-1. 공식 사이트 접속

아래 링크에서 "LTS"(왼쪽, 권장) 버전을 다운로드합니다.

Node.js 다운로드 페이지 열기 →
2-2. 설치 진행

다운로드된 설치 파일을 실행하고, 설치 마법사에서 "Next" 버튼만 계속 누르면 됩니다. 옵션은 기본값 그대로 두세요.

2-3. 설치 확인

명령 프롬프트(또는 터미널)를 열고 아래 명령어를 입력하세요.

⊞ Windows + R 키 → "cmd" 입력 후 엔터 → 검은 창이 뜨면:

C:\> node -v

⌘ + 스페이스 → "터미널" 검색 → 실행 후:

$ node -v

v20.x.x 같은 버전 번호가 나오면 성공입니다. ✅

3

MariaDB 설치

MariaDB는 데이터(고객, 리드, 원가 등)를 저장할 데이터베이스입니다.

3-1. 인스톨러 다운로드

아래 링크에서 "MSI Package" → 64-bit 를 다운로드하세요.

MariaDB 다운로드 →
3-2. 설치 마법사 진행

설치 중에 다음 두 화면이 가장 중요합니다:

  • "User settings" 화면:
    • "New root password" 에 비밀번호 입력 (꼭 메모하세요! 예: oci1234)
    • "Confirm" 에 같은 비밀번호 다시 입력
    • "Use UTF8 as default server's character set" 체크
  • "Database settings" 화면: 기본값(포트 3306) 그대로 두세요.
⚠ 비밀번호 메모 이 비밀번호는 잠시 후 .env 파일에 입력해야 합니다. 잊지 마세요!
3-3. 설치 확인 도구 - HeidiSQL

MariaDB 설치 시 HeidiSQL이라는 도구가 함께 설치됩니다 (시작 메뉴에서 검색). 이게 데이터베이스를 시각적으로 다룰 수 있게 해줍니다.

3-1. Homebrew 설치 (이미 있으면 생략)

터미널에서:

$ /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
3-2. MariaDB 설치 + 실행
$ brew install mariadb
$ brew services start mariadb
3-3. root 비밀번호 설정
$ sudo mysql_secure_installation

질문이 여러 개 나오는데, 다음과 같이 답하세요:

  • "Enter current password for root" → 그냥 Enter
  • "Switch to unix_socket authentication?" → n
  • "Set root password?" → Y → 비밀번호 입력 (꼭 메모!)
  • 나머지 질문들 → 모두 Y
4

프로젝트 파일 압축 해제

받으신 oci-crm.zip 파일을 적당한 위치에 풀어 주세요.

  • 추천 위치:
    • Windows: C:\oci-crm (C 드라이브 바로 아래)
    • macOS: ~/oci-crm (홈 폴더 아래)
  • 경로에 한글이나 공백이 들어가지 않게 주의하세요. (예: "내 문서" 폴더 ❌)
💡 잘 풀렸는지 확인 폴더 안에 package.json, server.js, schema.sql, public 폴더 등이 보이면 OK.
5

데이터베이스 생성 + 샘플데이터 로드

제공된 schema.sql 파일을 한번에 실행하면 테이블과 샘플 데이터가 모두 들어갑니다.

HeidiSQL을 이용하면 클릭만으로 가능합니다.

  1. HeidiSQL 실행 (시작 메뉴 검색)
  2. 새 세션 만들기:
    • 호스트: 127.0.0.1
    • 사용자: root
    • 암호: 3단계에서 메모한 비밀번호
    • "열기" 클릭
  3. SQL 파일 실행: 메뉴 파일 → SQL 파일 실행 → schema.sql 선택
  4. 좌측에 oci_power_crm 데이터베이스가 생기면 성공 ✅

터미널에서 압축 푼 폴더로 이동 후:

$ cd ~/oci-crm
$ mysql -u root -p < schema.sql

비밀번호를 묻습니다. 입력해도 글자가 보이지 않는 게 정상입니다. 입력 후 엔터.

Windows 명령 프롬프트에서도 동일하게 가능:

C:\> cd C:\oci-crm
C:\oci-crm> mysql -u root -p < schema.sql
"mysql is not recognized" 오류가 나면? MariaDB 설치 폴더의 bin 경로(예: C:\Program Files\MariaDB 11.x\bin\mysql.exe)를 직접 사용하거나, HeidiSQL을 이용하세요.
✅ 성공 확인 오류 없이 끝났다면 8개의 테이블 + 샘플 데이터 50여 건이 들어간 상태입니다.
6

환경 설정 파일(.env) 만들기

CRM 서버에게 "DB 비밀번호가 뭐야?"를 알려주는 파일입니다.

6-1. 메모장 또는 텍스트 편집기 열기

아무 메모장이나 좋습니다. (Windows 메모장 / macOS TextEdit / VS Code 등)

6-2. 다음 내용을 그대로 입력하세요
DB_HOST=localhost
DB_PORT=3306
DB_USER=root
DB_PASSWORD=여기에_3단계에서_정한_비밀번호
DB_NAME=oci_power_crm
PORT=3000

DB_PASSWORD 부분만 본인 비밀번호로 바꾸세요. 따옴표는 넣지 마세요.

6-3. 파일 저장 - 매우 중요!
  • 저장 위치: 프로젝트 폴더 (예: C:\oci-crm\)
  • 파일명: .env (앞에 점이 있고, 확장자는 없음)
  • 파일 형식: "모든 파일 (*.*)" 선택
  • 인코딩: UTF-8
⚠ Windows 메모장 주의사항 메모장이 자동으로 .env.txt로 저장할 수 있습니다. 파일 탐색기에서 보기 → "파일 확장명" 체크 후, 만약 .env.txt로 되어 있다면 .env로 이름 변경하세요.
7

의존성 설치 (필요한 부품 받아오기)

CRM이 작동하는 데 필요한 도구들을 자동으로 다운로드합니다. 인터넷 연결 필요.

7-1. 명령 프롬프트(터미널)에서 프로젝트 폴더로 이동
C:\> cd C:\oci-crm
$ cd ~/oci-crm
7-2. 설치 명령어 실행
npm install

⏱ 1~3분 정도 걸립니다. 화면에 글자가 막 흘러가도 정상입니다. 끝나면 다시 입력 대기 상태로 돌아옵니다.

경고(WARN) 메시지가 보여도 괜찮습니다. 빨간 ERROR가 안 나면 정상입니다.
8

서버 실행

드디어 CRM 서버를 켭니다!

npm start

아래와 비슷한 메시지가 나오면 성공입니다:

# OCI CRM 서버 실행중
# http://localhost:3000
# DB: oci_power_crm @ localhost:3306
⚠ 이 창은 닫지 마세요! 검은 창을 닫으면 서버가 꺼집니다. 나중에 종료하려면 그 창에서 Ctrl + C 를 누르세요.
9

브라우저에서 접속!

크롬, 엣지, 사파리 등 어떤 브라우저든 좋습니다.

주소창에 입력:

http://localhost:3000
🎉 축하합니다! OCI 로고가 보이는 CRM 화면이 나오면 성공입니다. 좌측 메뉴를 눌러 대시보드 / 파이프라인 / 영업리드 등을 둘러보세요.

같은 사내망의 다른 컴퓨터에서 접속하려면?

  1. 서버 컴퓨터의 IP 주소 확인:
    • Windows: cmd에서 ipconfig 입력 → "IPv4 주소"
    • macOS: 시스템설정 → 네트워크 → Wi-Fi → 세부사항
  2. 다른 컴퓨터 브라우저에서 http://[그_IP]:3000 입력 (예: http://192.168.0.10:3000)
  3. 방화벽이 막으면 "포트 3000 허용"을 추가해야 합니다.

막혔을 때 (자주 발생하는 문제)

대부분 아래에서 해결됩니다.

"node" 또는 "npm" 명령어를 못 찾는다고 나와요
Node.js가 제대로 설치되지 않았거나, 명령 프롬프트를 Node.js 설치 전에 열어둔 창이라서 그렇습니다.

해결: 명령 프롬프트(터미널)를 완전히 닫고 다시 여세요. 그래도 안 되면 Node.js를 다시 설치하세요.
"Access denied for user 'root'" 오류가 떠요
.env 파일의 DB_PASSWORD 값이 MariaDB 설치 시 정한 비밀번호와 다릅니다.

해결: .env 파일을 다시 열어 비밀번호를 정확히 입력하세요. 따옴표(")나 앞뒤 공백이 들어가면 안 됩니다.
"ECONNREFUSED 127.0.0.1:3306" 오류가 떠요
MariaDB 서비스가 꺼져 있습니다.

Windows 해결: 서비스(시작메뉴 검색) → "MariaDB" 항목을 찾아 우클릭 → "시작"
macOS 해결: 터미널에서 brew services start mariadb
"포트 3000이 이미 사용 중"이라고 나와요
다른 프로그램이 3000번 포트를 쓰고 있습니다.

해결: .env 파일에서 PORT=3000을 PORT=3001로 바꾸고, 브라우저에서도 http://localhost:3001로 접속.
".env 파일이 안 만들어지고 .env.txt로 저장돼요
Windows 메모장은 자동으로 .txt를 붙입니다.

해결 방법 1: 저장 시 "파일 형식"을 "모든 파일 (*.*)"로 변경하고 파일명을 .env로 입력.
해결 방법 2: 파일 탐색기에서 보기 → "파일 확장명" 체크 → .env.txt를 .env로 이름 변경.
화면은 떴는데 데이터가 비어 있어요
5단계(스키마 로드)가 누락된 것 같습니다.

해결: HeidiSQL이나 명령줄에서 schema.sql을 다시 실행했는지 확인하세요. 좌측 메뉴 "설정" 페이지의 "DB 상태 확인"을 눌러 연결과 데이터 건수를 점검할 수 있습니다.
다음 날 다시 시작하려면 어떻게 하나요?
  1. MariaDB가 자동 실행되는지 확인 (Windows 서비스에서 "자동" 으로 설정해 두세요)
  2. 명령 프롬프트(터미널)를 열고 cd C:\oci-crm (또는 cd ~/oci-crm)
  3. npm start 실행
  4. 브라우저에서 http://localhost:3000 접속

매번 명령어 입력이 귀찮다면: 프로젝트 폴더에 start.bat(Windows) 또는 start.command(macOS) 파일을 만들어 npm start 한 줄만 넣어두면, 더블클릭으로 실행됩니다.
모든 데이터를 처음 상태로 초기화하려면?
HeidiSQL 또는 터미널에서:
DROP DATABASE oci_power_crm;
그리고 다시 schema.sql을 실행하세요. 모든 데이터가 샘플 상태로 돌아옵니다.
